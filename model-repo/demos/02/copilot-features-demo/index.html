Code Completion Demo

Watch how GitHub Copilot suggests code as you type:

// Define a function to calculate the Fibonacci sequence
function fibonacci(n) {
    // Copilot will suggest the implementation here
}
✨ GitHub Copilot Suggestion 
function fibonacci(n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}

How It Works

GitHub Copilot analyzes the context of your code and comments to suggest relevant implementations. In this example, it recognized the need for a Fibonacci sequence implementation and provided a recursive solution.

Key points:

  • Suggestions appear as you type
  • Press Tab to accept or Esc to reject
  • Multiple suggestions may be available (Alt+[ or Alt+] to navigate)

Comment to Code Demo

See how detailed comments help Copilot generate better code:

// Create a function that validates an email address
// It should:
// 1. Check if the email contains an @ symbol
// 2. Verify there's text before and after the @ symbol
// 3. Ensure the domain has at least one period
// 4. Return true if valid, false otherwise
✨ GitHub Copilot Suggestion 
function validateEmail(email) {
    if (!email.includes('@')) return false;
    
    const [localPart, domain] = email.split('@');
    if (!localPart || !domain) return false;
    
    if (!domain.includes('.')) return false;
    
    return true;
}

How It Works

Detailed comments significantly improve the quality of Copilot's suggestions. By clearly stating requirements, you help Copilot understand exactly what you need.

Best practices for comments:

  • Be specific about requirements
  • Use numbered lists for multi-step processes
  • Include expected inputs and outputs
  • Mention edge cases to handle

Test Generation Demo

GitHub Copilot can help generate tests for your code:

// Function to test
function calculateDiscount(price, discountPercent) {
    if (discountPercent < 0 || discountPercent > 100) {
        throw new Error('Discount percentage must be between 0 and 100');
    }
    return price - (price * (discountPercent / 100));
}

// Generate unit tests for the calculateDiscount function
// Include tests for valid inputs and edge cases
✨ GitHub Copilot Suggestion 
// Test suite for calculateDiscount
describe('calculateDiscount', () => {
    test('applies correct discount to price', () => {
        expect(calculateDiscount(100, 20)).toBe(80);
        expect(calculateDiscount(50, 10)).toBe(45);
    });

    test('handles zero discount', () => {
        expect(calculateDiscount(100, 0)).toBe(100);
    });

    test('handles 100% discount', () => {
        expect(calculateDiscount(100, 100)).toBe(0);
    });

    test('throws error for negative discount', () => {
        expect(() => calculateDiscount(100, -10)).toThrow();
    });

    test('throws error for discount over 100%', () => {
        expect(() => calculateDiscount(100, 110)).toThrow();
    });
});

How It Works

GitHub Copilot can generate comprehensive test suites by analyzing your code's functionality. It identifies:

  • Normal use cases
  • Edge cases
  • Error conditions
  • Boundary values

This helps ensure your code is robust and handles all scenarios correctly.

Code Explanation Demo

GitHub Copilot can explain complex code:

// Explain what this code does
const memoize = (fn) => {
    const cache = {};
    return (...args) => {
        const key = JSON.stringify(args);
        if (key in cache) {
            return cache[key];
        }
        const result = fn.apply(this, args);
        cache[key] = result;
        return result;
    };
};
✨ GitHub Copilot Explanation

This code implements a memoization function. Memoization is an optimization technique that speeds up function calls by caching the results of expensive function calls.

Here's how it works:

  1. The memoize function takes another function fn as input
  2. It creates a cache object to store results
  3. It returns a new function that:
    • Converts its arguments to a string to use as a cache key
    • Checks if the result is already in the cache
    • If found, returns the cached result
    • If not found, calls the original function, stores the result, and returns it

This technique is particularly useful for computationally expensive functions or functions that are called repeatedly with the same inputs.

How It Works

GitHub Copilot can analyze code and provide clear explanations of what it does. This is useful for:

  • Understanding unfamiliar code
  • Documenting complex functions
  • Onboarding new team members
  • Learning new programming concepts
